# 경기테크노파크 규정 및 관계법령 가이드북 리디자인 설계

## 1. 목적

이 프로젝트의 핵심 목적은 경기테크노파크 내부 규정과 관계 법령을 RAG로 검색해,
사용자가 채팅으로 질문했을 때 근거 있는 답변을 빠르게 받도록 돕는 것이다.

이번 리디자인의 목표는 다음 세 가지다.

- 제품의 첫인상을 `운영 콘솔`이 아니라 `채팅 중심 규정/법령 가이드북`으로 전환한다.
- 메인 화면의 정보 밀도를 낮춰 첫 질문 전 인지 부담을 줄인다.
- 프로젝트명을 `경기테크노파크 규정 및 관계법령 가이드북`으로 전면 교체하고,
  공공기관형 단정함보다 한 단계 더 현대적인 정책/리서치 툴 톤으로 정리한다.

## 2. 현재 문제

현재 UI는 동작은 가능하지만, 메인 목적이 한눈에 보이지 않는다.

- 메인 화면에 사용자용 정보와 운영자용 정보가 섞여 있다.
- 사이드바에서 `지름길`, `세션 사용량`, `상태`, `법령 구성`이 동시에 경쟁한다.
- 카드, 배지, 박스가 많아서 실제 정보량보다 화면이 더 복잡하게 느껴진다.
- 법령/문서 탐색 구조가 메인 질문 경험보다 더 강하게 보인다.
- 제품명이 현재 목적과 톤을 충분히 설명하지 못한다.

정리하면, 문제는 기능 부족보다도 `첫 화면의 우선순위가 흐린 것`이다.

## 3. 확정된 방향

### 3.1 제품 포지셔닝

- 메인 성격: `직원 검색/질의 중심`
- 구조: `채팅 중심 + 보조 레일`
- 문서 관리: `보조 화면`
- 법령 추가: `사용자 셀프서비스 허용`
- 시각 톤: `정책/리서치 툴형 현대성`

### 3.2 새 프로젝트명

프로젝트명은 아래 이름으로 전면 교체한다.

`경기테크노파크 규정 및 관계법령 가이드북`

반영 범위:

- 메인 UI 헤더
- 브라우저 제목
- 채팅/관리자 화면 문구
- README 및 기본 프로젝트 소개 문구

## 4. 정보 구조

### 4.1 메인 화면

메인 화면은 질문을 시작하는 데 필요한 요소만 남긴다.

유지:

- 프로젝트명
- 한 줄 설명 또는 제품 소개
- 큰 질문 입력창
- 추천 질문 3~4개
- 답변 카드
- 최근 근거 보기
- 관계 법령 찾기
- 법령 추가 진입점

제거 또는 이동:

- 세션 사용량
- knowledge status
- 전체 문서 수 요약
- 확장형 법령 구성 목록
- 운영자 성격의 상태 패널

핵심 원칙은 `첫 화면에서 질문을 시작하는 데 직접 필요 없는 것은 메인에서 뺀다`이다.

### 4.2 보조 레일

메인 화면 오른쪽에는 얇은 보조 레일을 둔다. 이 영역은 메인 콘텐츠와 경쟁하지 않고,
질문 흐름을 지원하는 부가 기능만 제공한다.

구성:

- 관계 법령 빠른 찾기
- 최근 확인한 근거
- 법령 추가

보조 레일은 `탐색 패널`이지 `전체 정보 창고`가 아니다. 문서 운영 정보와 집계 정보는 넣지 않는다.

### 4.3 문서 관리 화면

문서 관리는 메인과 분리된 보조 화면으로 둔다.

원칙:

- 메인 화면에서는 운영자 관점 수치를 강조하지 않는다.
- 업로드, 재색인, 분류 변경, 삭제, 법령 수집 등 관리 기능은 여기로 이동한다.
- 사용자가 직접 법령을 추가하는 흐름도 메인에서는 간단한 진입점만 제공하고,
  상세 동작은 보조 화면 또는 모달에서 처리한다.

## 5. 시각 시스템

### 5.1 전체 인상

현재의 `짙은 사이드바 + 다수의 입체 카드` 구조를 줄이고, 더 조용하고 현대적인 리서치 툴 느낌으로 바꾼다.

방향:

- 밝은 뉴트럴 배경
- 얕은 질감 또는 미세한 그리드만 사용
- 포인트 컬러는 한 축으로 제한
- 장식용 박스보다 읽기 흐름을 우선

### 5.2 컬러

색상 시스템은 공공기관형 신뢰감을 유지하되 더 제품적으로 정리한다.

원칙:

- 포인트 컬러는 `딥 블루` 또는 `티얼` 축으로 통일
- 경고/오류/성공만 예외 색상 허용
- 배경, 경계선, 강조색을 CSS 변수로 정리
- 장식용 따뜻한 그라데이션, 과한 베이지 계열 겹침은 줄인다

### 5.3 타이포그래피

타이포는 `제목의 존재감`과 `본문의 가독성`을 분리한다.

원칙:

- 프로젝트명은 존재감 있는 한국어 제목 스타일
- 본문/인터페이스는 읽기 좋은 산세리프
- 제목, 설명, 보조 문구의 계층을 명확히 구분
- 카드 수를 줄이는 대신 활자 위계로 정보 구조를 만든다

### 5.4 컴포넌트 톤

현재의 둥근 배지와 둥근 카드가 많은 구조를 더 절제한다.

원칙:

- 버튼, 칩, 입력창의 반경을 약간 줄인다
- 그림자는 약하게, 경계선은 더 명확하게
- 메인 답변 카드와 보조 근거 카드의 위계를 분명히 나눈다
- 시선 분산을 유발하는 장식용 컴포넌트는 제거한다

## 6. 경험 설계

### 6.1 첫 화면

첫 화면에서 사용자가 느껴야 하는 순서는 아래와 같다.

1. 여기는 규정/법령을 채팅으로 물어보는 곳이다.
2. 바로 질문할 수 있다.
3. 답변은 근거와 함께 제공된다.
4. 필요하면 관계 법령과 원문을 곧바로 확인할 수 있다.

이 흐름을 방해하는 모든 요소는 메인에서 후순위로 내린다.

### 6.2 답변 신뢰감

RAG 제품이므로 답변의 신뢰감 표현이 중요하다.

강화 요소:

- 답변 카드 상단의 근거성 설명 배지
- 최근 근거 영역의 간결한 요약
- 원문 또는 위치 확인 동선 단순화

약화 요소:

- 지나치게 많은 상태 수치
- 근거 카드보다 시각적으로 더 강한 운영 카드

### 6.3 법령 추가

사용자가 API 기반으로 법령을 추가할 수 있어야 한다.

원칙:

- 메인에서도 `법령 추가` 진입은 보인다
- 하지만 입력 폼 전체를 메인에 상시 노출하지는 않는다
- 사용 흐름은 간단하고 자기설명적이어야 한다

## 7. 구현 범위

이번 리디자인에서 구현 대상으로 본다.

- 채팅 메인 레이아웃 재구성
- 사이드바/보조 레일 역할 재정의
- 메인 화면에서 운영자 수치 제거 또는 이동
- 프로젝트명 전면 교체
- 시각 토큰 재정의
- 버튼/칩/카드/패널 스타일 단순화
- 문서 관리 화면의 보조 화면화

이번 단계에서 과도한 신규 기능 추가는 하지 않는다.

- 새로운 대규모 검색 시스템
- 완전한 IA 재설계 이상의 기능 확장
- 인증/권한 체계 변경

## 8. 수용 기준

리디자인 완료 판단 기준은 아래와 같다.

- 첫 화면에서 질문 입력이 가장 강하게 보인다.
- 메인 화면에서 운영 지표가 사라지거나 충분히 후순위로 이동한다.
- `경기테크노파크 규정 및 관계법령 가이드북` 이름이 UI와 문서에 일관되게 반영된다.
- 전체 톤이 내부 콘솔보다 정책/리서치 툴에 가깝게 보인다.
- 법령 추가는 사용자가 스스로 접근 가능한 위치에 남아 있다.
- 메인 화면의 카드/배지/섹션 수가 줄어 복잡도가 체감상 낮아진다.

## 9. 구현 메모

- 현재 `Shell` 중심 구조에서 사이드바 역할을 재정의하는 것이 핵심이 될 가능성이 높다.
- `App`, `Shell`, 채팅 패널, 전역 스타일 파일이 주요 수정 지점이 될 가능성이 크다.
- README와 브라우저 메타 타이틀도 함께 반영해야 한다.
