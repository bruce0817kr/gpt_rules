# 내부 공개 체크리스트

## 1. 접근 통제

- [ ] `https://ai.gtp.or.kr/chat/`가 사내망, VPN, 리버스 프록시 allowlist 등 내부 경계 안에서만 열리는지 확인
- [ ] 관리자 기능(`문서 업로드`, `법령명으로 바로 추가`, `분류 변경`) 경로가 일반 사용자와 분리된 운영 정책으로 관리되는지 확인
- [ ] 운영 담당자/디지털전환팀 연락 체계가 공지되었는지 확인

## 2. 데이터/문서 상태

- [ ] 핵심 내부 문서(정관, 규정, 지침) 최신본이 적재되었는지 확인
- [ ] `Docs/MD/law_md` 기준 법령 Markdown 동기화가 완료되었는지 확인
- [ ] 법령 추가 API 또는 배치 동기화 후 `law` 카테고리 문서 수가 기대값과 일치하는지 확인
- [ ] 테스트용 문서, 중복 문서, 잘못 적재된 문서가 남아 있지 않은지 확인

## 3. 검색/답변 품질

- [ ] 대표 질문 10개 이상으로 검색 품질 확인
- [ ] citation 클릭 시 source viewer가 열리고, 관련 section과 snippet이 강조되는지 확인
- [ ] 표/별표 기반 답변(예: 경조사 휴가, 수당, 여비)이 실제 표/목록 형태로 응답되는지 확인
- [ ] 잘못된 근거 문서를 참조하는 사례가 없는지 샘플링 점검

## 4. 운영 기능

- [ ] 다중 파일 업로드 정상 동작 확인
- [ ] 문서 분류 수동 변경 후 재색인해도 분류가 유지되는지 확인
- [ ] `문서 구성` 검색(제목/파일명/본문)과 shortcut 검색 모달이 정상 동작하는지 확인
- [ ] 법령명으로 바로 추가 기능이 정상 동작하는지 확인

## 5. 배치/동기화

- [ ] `scripts/run_law_sync.ps1` 수동 실행 확인
- [ ] `scripts/run_law_sync.bat` 실행 확인
- [ ] 필요 시 `scripts/register_law_sync_task.ps1`로 예약 작업 등록
- [ ] `scripts/logs/`에 동기화 로그가 생성되는지 확인

## 6. 시스템 상태

- [ ] `http://localhost:8000/api/health` 정상
- [ ] `http://localhost:8088/chat/api/health` 정상
- [ ] Qdrant가 정상 기동 중인지 확인
- [ ] KoE5 / reranker warm-up 이후 초기 질의 응답 시간이 허용 범위인지 확인

## 7. 공개 전 최종 확인

- [ ] 운영 담당자에게 첫 공개 범위(파일럿 사용자/부서) 공유
- [ ] 오류 신고 경로 공지
- [ ] 공개 후 첫 주 모니터링 담당자 지정

## 운영 연락

- 기능 추가 및 오류 신고: 디지털전환팀 이도훈
